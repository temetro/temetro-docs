ይህ መመሪያ Temetro ን ለግምገማ፣ ለስልጠና ወይም ለአነስተኛ ውስጣዊ ማሰማሪያዎች በአንድ ማሽን ላይ ለማስኬድ በጣም ፈጣን መንገድን ያሳያል።

## የሚያስፈልጉ ነገሮች

ከመጀመርዎ በፊት ማሽኑ ይህን እንዳለው ያረጋግጡ፦

- Docker ተጭኗል
- Docker Compose v2 በ `docker compose` ትእዛዝ ይገኛል
- Git ተጭኗል
- ቢያንስ 2 GB RAM አለ
- `3000`፣ `3001`፣ `4000` እና `5432` ፖርቶች ነጻ ናቸው

## 1. የማሰማሪያ ሪፖዚቶሪውን ኮሎን ያድርጉ

```bash
git clone https://github.com/temetro/temetro-deploy
cd temetro-deploy
```

የእርስዎ የIT ቡድን በአንድ ሰርቨር ላይ ብዙ የሆስፒታል ስርዓቶችን ካስተዳደረ፣ ለመጠባበቂያና ለጥገና ቀላል እንዲሆን በመጀመሪያ ለTemetro የተለየ ፎልደር ይፍጠሩ።

## 2. የምሳሌ የአካባቢ ፋይሉን ይቅዱ

```bash
cp .env.example .env
```

ይህ የሚሰራ የአዋቅር ፋይልዎን ይፈጥራል። ከመጀመሪያው አሂድ በፊት ጥቂት እሴቶችን ብቻ ማስተካከል ያስፈልጋል።

## 3. አስፈላጊውን አነስተኛ እሴቶች ያስቀምጡ

`.env` ን በጽሁፍ አርታዒ ውስጥ ክፈቱ እና እነዚህን ያዘምኑ፦

- `POSTGRES_PASSWORD`
- `APP_URL`
- `BETTER_AUTH_SECRET`

የauth ሚስጥሩን በዚህ ይፍጠሩ፦

```bash
openssl rand -base64 32
```

## 4. Temetro ን ያስጀምሩ

```bash
docker compose up -d
```

Docker የሚያስፈልጉትን ምስሎች ያወርዳል፣ የ PostgreSQL የውሂብ ጎታ ኮንቴነር ይፈጥራል፣ እና የዌብ ፣ የውስጥ API እና የሰነዶች አገልግሎቶችን ያስጀምራል።

## 5. አገልግሎቶቹ እየሰሩ መሆናቸውን ያረጋግጡ

```bash
docker compose ps
```

እነዚህን ኮንቴነሮች ማየት አለብዎት፦

- `postgres`
- `api`
- `web`
- `docs`

## 6. Temetro ን በአሳሽዎ ውስጥ ይክፈቱ

- ዳሽቦርድ: `http://localhost:3000`
- ሰነዶች: `http://localhost:3001/docs`

በርቀት ሰርቨር ላይ እየጫኑ ከሆነ፣ `localhost` ን በሰርቨሩ IP አድራሻ ወይም በውስጣዊ hostname ይተኩ።

## ቀጣይ ምን ማድረግ አለብዎት

ሁሉንም የአካባቢ እሴቶች ለመገምገም ወደ [አዋቅር](./configuration) ይቀጥሉ፣ ከዚያም የመጀመሪያውን አስተዳዳሪ መለያ ለማዘጋጀት እና መጫኑን ለማረጋገጥ በ [የመጀመሪያ አሂድ](./first-run) ይጨርሱ።
