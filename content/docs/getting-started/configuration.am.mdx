Temetro ቀላል እንዲቆይ ተዘጋጅቷል። አብዛኛዎቹ ጭነቶች በደህና ለመጀመር ጥቂት የአካባቢ እሴቶችን ብቻ ያስፈልጋቸዋል።

## ዋና የአካባቢ ተለዋዋጮች

ዋና ተለዋዋጮቹ በማሰማሪያ ሪፖዚቶሪው `.env` ፋይል ውስጥ ይገኛሉ።

| ተለዋዋጭ | ያስፈልጋል | ምሳሌ | ዓላማ |
| --- | --- | --- | --- |
| `POSTGRES_PASSWORD` | አዎ | `strong-password-here` | ለ PostgreSQL የውሂብ ጎታ ተጠቃሚ የይለፍ ቃል |
| `APP_URL` | አዎ | `http://localhost:3000` | ዳሽቦርዱ የሚጠቀምበት የሕዝብ URL |
| `BETTER_AUTH_SECRET` | አዎ | የተፈጠረ እሴት | የማረጋገጫ ውሂብን ለመፈረም እና ለመጠበቅ የሚያገለግል ሚስጥር |
| `SMTP_HOST` | አማራጭ | `smtp.example.org` | ለወጪ ኢሜይል የደብዳቤ አገልጋይ ስም |
| `SMTP_PORT` | አማራጭ | `587` | የደብዳቤ አገልጋይ ፖርት |
| `SMTP_USER` | አማራጭ | `alerts@example.org` | ለ SMTP ማረጋገጫ የተጠቃሚ ስም |
| `SMTP_PASS` | አማራጭ | `mail-password` | ለ SMTP ማረጋገጫ የይለፍ ቃል |

## የመጀመሪያ የሚመከሩ ቅንብሮች

ለሙከራ አካባቢ፦

- `APP_URL=http://localhost:3000` ያስቀምጡ
- ጠንካራ ነገር ግን ለጊዜው ለማስታወስ ቀላል የሆነ የውሂብ ጎታ የይለፍ ቃል ይጠቀሙ
- አዲስ `BETTER_AUTH_SECRET` ይፍጠሩ

ለእውነተኛ የሆስፒታል አካባቢ፦

- ሰራተኞች በእውነት የሚጠቀሙበትን ውስጣዊ DNS ስም ወይም የሕዝብ domain ይጠቀሙ
- `.env` ን የፋይል ፍቃዱ የተገደበ ደህንነቱ የተጠበቀ ቦታ ውስጥ ያስቀምጡ
- ለአደጋ ጊዜ መመለስ `BETTER_AUTH_SECRET` የተደህነ ቅጂ ይያዙ

## የውሂብ ጎታ ግንኙነት

የዌብ አገልግሎቱ `DATABASE_URL` ን በ Docker ውስጥ ያሉ እሴቶችን በመጠቀም በራሱ ይገነባል። ብዙ ጊዜ የመተግበሪያ ደረጃ የውሂብ ጎታ URL በእጅ ማስተካከል አያስፈልግም።

ከጊዜ በኋላ ወደ ውጫዊ PostgreSQL አገልጋይ ከተዛወሩ፣ ዳሽቦርዱ ከአዲሱ የውሂብ ጎታ ጋር በደህና እንዲገናኝ በማሰማሪያ ፋይሎችዎ ውስጥ ያለውን host እና credentials ያዘምኑ።

## ለሆስፒታል IT ቡድኖች የኔትወርክ ምክር

ሆስፒታልዎ የአካባቢ ኔትወርክ ካለው ነገር ግን ውስን የኢንተርኔት ግንኙነት ካለው፦

- Temetro ን በቦታው ላይ በሚቆይ ማሽን ላይ ያስተናግዱ
- የተረጋጋ IP አድራሻ ወይም ውስጣዊ DNS ስም ይስጡት
- ሰራተኞች እንደ እርስዎ ቅንብር ፖርት `80` ወይም `3000` ላይ መድረስ እንደሚችሉ ያረጋግጡ
- ከህንፃ ውጭ ወይም በርቀት መድረሻ ከመፍቀድዎ በፊት HTTPS በ NGINX ይጠቀሙ

## መጠባበቂያዎች

Temetro ጠቃሚ የአሰራር ውሂብን በ PostgreSQL ውስጥ ያከማቻል። ከመጀመሪያው ቀን ጀምሮ የውሂብ ጎታ መጠባበቂያዎችን የመደበኛ ሂደትዎ አካል ያድርጉ።

ቢያንስ እነዚህን ያስቀምጡ፦

- የ PostgreSQL ውሂብ
- `.env` ፋይል
- ማንኛውም ብጁ reverse-proxy ወይም SSL ሰርቲፍኬት ፋይሎች
