يوضح هذا الدليل أسرع طريقة لتشغيل Temetro على جهاز واحد لأغراض التقييم أو التدريب أو النشر الداخلي الصغير.

## المتطلبات

قبل أن تبدأ، تأكد من أن الجهاز يحتوي على:

- Docker مثبت
- Docker Compose v2 متاح عبر الأمر `docker compose`
- Git مثبت
- ذاكرة RAM لا تقل عن 2 جيجابايت
- المنافذ `3000` و`3001` و`4000` و`5432` متاحة

## 1. استنساخ مستودع النشر

```bash
git clone https://github.com/temetro/temetro-deploy
cd temetro-deploy
```

إذا كان فريق تقنية المعلومات يدير عدة أنظمة مستشفى على نفس الخادم، فأنشئ مجلدًا مخصصًا لـ Temetro أولًا حتى يسهل نسخه احتياطيًا وصيانته.

## 2. نسخ ملف البيئة المثال

```bash
cp .env.example .env
```

سيؤدي ذلك إلى إنشاء ملف الإعدادات العامل. ستحتاج فقط إلى تعديل بعض القيم قبل أول تشغيل.

## 3. ضبط القيم المطلوبة الدنيا

افتح ملف `.env` في محرر نصوص وحدّث ما يلي:

- `POSTGRES_PASSWORD`
- `APP_URL`
- `BETTER_AUTH_SECRET`

أنشئ سر المصادقة باستخدام:

```bash
openssl rand -base64 32
```

## 4. تشغيل Temetro

```bash
docker compose up -d
```

سيقوم Docker بتنزيل الصور المطلوبة وإنشاء حاوية قاعدة بيانات PostgreSQL وتشغيل خدمات الويب وواجهة API الداخلية والتوثيق.

## 5. تأكيد أن الخدمات تعمل

```bash
docker compose ps
```

يجب أن ترى حاويات للخدمات التالية:

- `postgres`
- `api`
- `web`
- `docs`

## 6. افتح Temetro في المتصفح

- لوحة التحكم: `http://localhost:3000`
- التوثيق: `http://localhost:3001/docs`

إذا كنت تثبّت النظام على خادم بعيد، فاستبدل `localhost` بعنوان IP الخاص بالخادم أو باسم المضيف الداخلي.

## ماذا بعد؟

تابع إلى [الإعدادات](./configuration) لمراجعة جميع قيم البيئة، ثم أكمل مع [التشغيل الأول](./first-run) لإعداد أول حساب إداري والتحقق من التثبيت.
