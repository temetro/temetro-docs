ይህ ገጽ Temetro በትክክል መጀመሩን እና ቡድንዎ በደህና መጠቀም መጀመር እንደሚችል ለማረጋገጥ ያግዛል።

## 1. የኮንቴነር ጤናን ያረጋግጡ

ይህን ያሂዱ፦

```bash
docker compose ps
```

ሁሉም ዋና አገልግሎቶች ጤናማ ወይም እየሰሩ እንዳሉ መግለጽ አለባቸው።

የ PostgreSQL አገልግሎቱ አሁንም እየተጀመረ ከሆነ፣ ትንሽ ይጠብቁ እና ትእዛዙን እንደገና ያሂዱ።

## 2. ዳሽቦርዱን ይክፈቱ

በአሳሽዎ ውስጥ የዳሽቦርዱን URL ይጎብኙ፦

```text
http://localhost:3000
```

ወደ የመግቢያ ገጹ መድረስ አለብዎት።

## 3. የመጀመሪያውን አስተዳዳሪ መለያ ይፍጠሩ

የመጀመሪያውን የሆስፒታል አስተዳዳሪ ለመመዝገብ በመግቢያ ማያ ገጹ ላይ ያለውን **Create an account** አገናኝ ይጠቀሙ። የአስፈጻሚ ቡድንዎ ወይም የሆስፒታል IT መሪው የሚቆጣጠረውን የኢሜይል አድራሻ ይምረጡ።

የሚመከረው የመጀመሪያ አድሚን መገለጫ፦

- ስም: የስርዓቱ መውጫ ወይም ክትትል ተጠያቂ ሰው
- ኢሜይል: ረጅም ጊዜ የሚጠቀሙበት የድርጅት ኢሜይል አድራሻ
- የይለፍ ቃል: በሆስፒታሉ የይለፍ ቃል አስተዳዳሪ ውስጥ የሚቀመጥ ጠንካራ የቃል ሐረግ

## 4. የመጀመሪያውን ዳሽቦርድ ይመልከቱ

ከገቡ በኋላ ዋና ክፍሎቹ በትክክል እንደሚጫኑ ያረጋግጡ፦

- የዳሽቦርድ አጠቃላይ እይታ
- ታካሚዎች
- ቀጠሮዎች
- ሰራተኞች
- ቅንብሮች

በዚህ የመሠረት ደረጃ አንዳንድ ውሂብ አቀማመጥንና የስራ ሂደቶችን ለማረጋገጥ የሚያገለግል የማሳያ ውሂብ ሊሆን ይችላል።

## 5. ካስፈለገ ውሂብ ይሙሉ ወይም ያስገቡ

ለግምገማ የማሳያ መዝገቦች ከፈለጉ፣ የseed ትእዛዙን ከ monorepo ወይም ከ web ኮንቴነር ያሂዱ፦

```bash
npm run seed
```

ወይም በ Docker ውስጥ፦

```bash
docker compose exec web npm run seed
```

## 6. ለእጅ ማስረከብ ይዘጋጁ

ክሊኒካዊ ሰራተኞች ሙከራ ከመጀመራቸው በፊት ይህን ያረጋግጡ፦

- የመግቢያ URL መዝግበው ይያዙ
- የመጀመሪያውን አስተዳዳሪ መረጃ ማን እንደሚይዝ ይመዝግቡ
- መጠባበቂያዎች የሚቀመጡበትን ቦታ ይወስኑ
- ለዝማኔዎች እና ለሰርቨር ክትትል ማን ተጠያቂ እንደሆነ ያረጋግጡ

## ቀጣይ ምን ማድረግ አለብዎት

ስርዓቱ በአካባቢዎ ላይ እየሰራ ከሆነ፣ የበለጠ የሚጠናከር የሰርቨር ማሰማሪያ ለማቀድ ወደ በራስ-ማስተናገድ ክፍል ይቀጥሉ።
