---
title: التشغيل الأول
description: تحقق من تثبيت Temetro الجديد وأعد سير عمل أول مسؤول في المستشفى.
---

تساعدك هذه الصفحة على التأكد من أن Temetro بدأ بشكل صحيح وأن فريقك يمكنه البدء باستخدامه بأمان.

## 1. افحص حالة الحاويات

شغّل:

```bash
docker compose ps
```

يجب أن تظهر جميع الخدمات الأساسية بحالة سليمة أو قيد التشغيل.

إذا كانت خدمة PostgreSQL ما تزال تعيد التشغيل، فانتظر قليلًا ثم أعد تشغيل الأمر.

## 2. افتح لوحة التحكم

انتقل إلى رابط لوحة التحكم في المتصفح:

```text
http://localhost:3000
```

يجب أن تصل إلى صفحة تسجيل الدخول.

## 3. أنشئ أول حساب إداري

استخدم رابط **Create an account** في شاشة تسجيل الدخول لتسجيل أول مسؤول للمستشفى. اختر عنوان بريد إلكتروني تابعًا لفريق التنفيذ أو لمسؤول تقنية المعلومات في المستشفى.

الملف الموصى به للمسؤول الأول:

- الاسم: الشخص المسؤول عن الإطلاق أو الإشراف
- البريد الإلكتروني: عنوان مؤسسي طويل الأمد
- كلمة المرور: عبارة مرور قوية محفوظة في مدير كلمات مرور المستشفى

## 4. راجع لوحة التحكم الأولى

بعد تسجيل الدخول، تأكد من أن الأقسام الرئيسية تعمل بشكل صحيح:

- نظرة عامة على لوحة التحكم
- المرضى
- المواعيد
- الموظفون
- الإعدادات

في مرحلة الهيكل الأولي، تكون بعض البيانات بيانات تجريبية تستخدم لتأكيد التخطيط ومسارات العمل.

## 5. أنشئ بيانات تجريبية أو نفّذ ترحيلًا عند الحاجة

إذا أردت سجلات تجريبية للتقييم، شغّل أمر التهيئة من المستودع الأحادي أو من حاوية الويب:

```bash
npm run seed
```

أو داخل Docker:

```bash
docker compose exec web npm run seed
```

## 6. استعد للتسليم

قبل أن يبدأ الموظفون السريريّون الاختبار، تأكد من أنك:

- وثّقت رابط تسجيل الدخول
- سجّلت من يحتفظ ببيانات اعتماد أول مسؤول
- حددت مكان تخزين النسخ الاحتياطية
- أكدت من المسؤول عن التحديثات ومراقبة الخادم

## ماذا بعد؟

إذا كان النظام يعمل محليًا، فانتقل إلى قسم الاستضافة الذاتية للتخطيط لنشر خادم أكثر استدامة.
