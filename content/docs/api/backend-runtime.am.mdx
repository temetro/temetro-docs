Temetro አሁን የዳሽቦርድ backend ን በአንድ አካል ፈንታ በሁለት ክፍሎች ያስኬዳል።

## የአሁኑ የአሂድ ክፍፍል

- Next.js የዳሽቦርድ በይነገጽን ያቀርባል
- Better Auth አሁንም በ Next.js ውስጥ ይሰራል
- የውስጥ Express አገልግሎት የዳሽቦርድ ውሂብን ያቀርባል

ይህ ለተጠቃሚው የሚታየውን የድር መተግበሪያ እንዲረጋ ሲያደርግ የዳሽቦርድ ውሂብ ኮንትራቶችን ወደ ተለየ backend አገልግሎት ያንቀሳቅሳል።

## የውስጥ መንገዶች

የ Express አገልግሎቱ አሁን ይህን ያቀርባል፦

- `GET /health`
- `GET /dashboard/home/:role`

እነዚህ መንገዶች ለውስጣዊ አገልግሎት-እርስ-በርስ ጥሪ ናቸው፣ ለአጠቃላይ የብራውዘር መዳረሻ አይደሉም።

## አካባቢ እና የአገልግሎት ግንኙነት

የ Next.js ዳሽቦርዱ ከ Express አገልግሎት በዚህ ይነበባል፦

- `INTERNAL_API_URL`

በ Docker Compose ውስጥ የተለመደው ዋጋ ይህ ነው፦

```text
http://api:4000
```

## Temetro ይህን ክፍፍል ለምን ይጠቀማል

ይህ ደረጃ Temetro ን ይህን ይሰጣል፦

- ለዳሽቦርድ ውሂብ ኮንትራቶች የበለጠ ግልጽ ድንበር
- ሙሉ የauth ዳግም ጽሁፍ ሳያስፈልግ የbackend ሎጂክን ለማሳደግ ቦታ
- ወደፊት API ለማስፋፋት ቀላል መንገድ

እስካሁን ድረስ auth እና የገጽ መንገድ አስተዳደር በ Next.js ውስጥ ይቀራሉ ስለዚህ ለውጡ በቁጥጥር ውስጥ ይቆያል።
