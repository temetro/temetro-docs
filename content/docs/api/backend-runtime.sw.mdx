Temetro sasa inaendesha backend ya dashibodi katika tabaka mbili badala ya moja.

## Mgawanyo wa sasa wa uendeshaji

- Next.js hutoa kiolesura cha dashibodi
- Better Auth bado inaendesha ndani ya Next.js
- huduma ya ndani ya Express hutoa data ya dashibodi

Hii huweka programu ya wavuti inayoonekana kwa mtumiaji ikiwa thabiti huku ikihamisha mikataba ya data ya dashibodi kwenda kwenye huduma maalum ya backend.

## Njia za ndani

Huduma ya Express kwa sasa hutoa:

- `GET /health`
- `GET /dashboard/home/:role`

Njia hizi zimekusudiwa matumizi ya ndani kati ya huduma, si kwa ufikiaji wa moja kwa moja wa umma kupitia kivinjari.

## Mazingira na uunganishaji wa huduma

Dashibodi ya Next.js husoma kutoka huduma ya Express kupitia:

- `INTERNAL_API_URL`

Katika Docker Compose thamani ya kawaida ni:

```text
http://api:4000
```

## Kwa nini Temetro inatumia mgawanyo huu

Awamu hii inaipa Temetro:

- mpaka ulio wazi zaidi wa mikataba ya data ya dashibodi
- nafasi ya kukuza mantiki ya backend bila kuandika upya uthibitishaji wote
- njia rahisi ya kupanua API baadaye

Uthibitishaji na uelekezaji wa kurasa bado vinabaki ndani ya Next.js kwa sasa ili mabadiliko yabaki kudhibitiwa.
