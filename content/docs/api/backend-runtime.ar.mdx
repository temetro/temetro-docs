أصبح Temetro الآن يشغّل الواجهة الخلفية للوحة التحكم عبر طبقتين بدلًا من طبقة واحدة.

## التقسيم الحالي لبيئة التشغيل

- يقوم Next.js بعرض واجهة لوحة التحكم
- ما زال Better Auth يعمل داخل Next.js
- توفّر خدمة Express داخلية حمولة بيانات لوحة التحكم

يسمح ذلك بالحفاظ على استقرار تطبيق الويب الظاهر للمستخدمين مع نقل عقود بيانات لوحة التحكم إلى خدمة خلفية مخصصة.

## المسارات الداخلية

توفّر خدمة Express حاليًا:

- `GET /health`
- `GET /dashboard/home/:role`

هذه المسارات مخصّصة للاستخدام الداخلي بين الخدمات، وليست للوصول العام المباشر من المتصفح.

## البيئة وربط الخدمات

تقرأ لوحة تحكم Next.js من خدمة Express عبر:

- `INTERNAL_API_URL`

وفي Docker Compose تكون القيمة المعتادة:

```text
http://api:4000
```

## لماذا يستخدم Temetro هذا التقسيم

يمنح هذا Temetro في هذه المرحلة:

- حدًا أوضح لعقود بيانات لوحة التحكم
- مساحة لتوسيع منطق الواجهة الخلفية دون إعادة كتابة كاملة للمصادقة
- مسارًا أبسط لتوسيع واجهات API لاحقًا

أما المصادقة وتوجيه الصفحات فما زالا داخل Next.js في الوقت الحالي حتى تبقى التغييرات مضبوطة.
