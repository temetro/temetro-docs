Temetro ለመጀመሪያው የማረጋገጫ ንብርብር Better Auth ይጠቀማል።

## አሁን ያለው የማረጋገጫ ሞዴል

ይህ መሠረታዊ አወቃቀር የሚያስችለው፦

- በኢሜይል እና በይለፍ ቃል መግባት
- በsession ላይ የተመሰረተ ማረጋገጫ
- የተጠበቁ የዳሽቦርድ መስመሮች
- በመተግበሪያ shell ውስጥ የserver-side session ምርመራ

## የመስመር አቀማመጥ

ዳሽቦርዱ Better Auth ን በዚህ መስመር ስር ያቀርባል፦

```text
/api/auth/[...all]
```

ይህ ለ Next.js App Router ግንኙነት የሚመከረው መደበኛ የመስመር ቅርጽ ነው።

## አስፈላጊ የአካባቢ ተለዋዋጮች

- `BETTER_AUTH_SECRET`
- `BETTER_AUTH_URL`
- `NEXT_PUBLIC_APP_URL`
- `DATABASE_URL`

## የsession ቆይታ

Temetro በአሁኑ ጊዜ ሰባት ቀን የsession መስኮት ይጠቀማል። ይህ የመደበኛ ሰራተኛ የስራ ሂደቶችን ለማስተናገድ በቂ ሲሆን መግቢያዎችም ውስን እንዲቆዩ ያደርጋል።

## የደህንነት ማስታወሻዎች

ለፕሮዳክሽን አጠቃቀም የሚከተሉትን በጥንቃቄ ይመልከቱ፦

- ልዩ የauth ሚስጥር ይፍጠሩ
- ሰርቨርዎን በ HTTPS ይጠብቁ
- የውሂብ ጎታ መዳረሻን ለታመኑ ኔትወርኮች ብቻ ይገድቡ
- የአስተዳዳሪ መለያዎችን ማን ሊፈጥር ወይም ሊያስተዳድር እንደሚችል ይወስኑ

## ወደፊት ሊጨመሩ የሚችሉ ባህሪያት

የ Better Auth መሠረት በኋላ ለእነዚህ ዓይነት ባህሪያት ሊሰፋ ይችላል፦

- የኢሜይል ማረጋገጫ
- የይለፍ ቃል መመለስ
- ሁለት-ደረጃ ማረጋገጫ
- ድርጅቶች እና ዝርዝር ፍቃዶች

እነዚህ ችሎታዎች ያለ ሙሉ የማረጋገጫ ክፍሉን እንደገና ማደስ አሁን ካለው ስታክ ጋር በቀላሉ ይጣጣማሉ።
