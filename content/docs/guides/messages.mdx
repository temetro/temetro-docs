The **Messages** page is a private, real-time chat for your clinic's staff — direct
messages and group conversations, delivered instantly.

## Messaging your team

### Starting a conversation

1. Open **Messages** and click **New conversation**.
2. Pick one or more clinic members. Choosing one person starts a **direct message**
   (if you already have a DM with them, temetro reopens it instead of creating a
   duplicate). Choosing several creates a **group**, which you can name.
3. Type and send. Messages arrive for the other person immediately — no refresh needed.

### Attaching files and appointments

Next to the message box, the **+** button lets you attach:

- **Files** — pick one or more files from your device (up to 10MB each). They upload
  and appear as chips above the box; recipients see a tap-to-download chip in the thread.
- **Appointments** — search by patient name and pick an appointment to share. It's sent
  as a tidy **appointment card** (name, date/time, type, provider) — a snapshot, so it
  still reads correctly even if the appointment changes later.

You can send attachments with or without a text message. Remove a staged attachment with
its **×** before sending.

### Staying on top of things

- **Unread badges** on conversations show exactly how many new messages are waiting;
  opening a conversation marks it read. The **Unread** filter narrows the inbox to
  just those conversations.
- In a thread, messages render as **chat bubbles** (yours on the right, theirs on the
  left) grouped by **day** (Today / Yesterday / date), with consecutive messages from the
  same person bundled under one name and timestamp — so long conversations stay easy to
  scan. Shared files and appointments appear as compact **attachment cards** in the thread.
- **Notifications** appear in the app when you receive a message while looking at
  another page.
- A **call button** sits to the left of each conversation in the inbox — click it to jump
  to [Meetings](/docs/guides/meetings) with that person preselected, ready to schedule or
  ring them in.

### System messages

Sometimes temetro itself sends you a message — for example, when an employee asks to
reset their password but no [email provider](/docs/guides/email-setup) is set up, the
clinic's admins get a **System** notice so they can step in. These conversations are easy
to spot: they show a **System** badge and a shield icon instead of a person's initials.

A System conversation is **read-only** — there's no message box and no call button, because
there's no one on the other end to reply to. Some System messages are **actionable**: a
password-reset notice, for instance, links straight to that employee in **Care Team** so an
admin can set a new password for them.

<Callout type="info" title="Clinic-only">
  Conversations stay within the active clinic, and only the participants of a
  conversation can read it — messaging is not visible to admins or anyone else.
</Callout>
