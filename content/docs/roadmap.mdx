temetro is open source and under active development. This page is the honest summary of
where things stand.

## In progress / planned

### Real AI chat

The chat UI is fully built, but free-form messages currently get a **mock reply** — no
language model is connected. The next planned step is a `/chat` endpoint on the backend
that sends your question (plus the relevant patient context) to an AI model and streams
the answer back into the same card-based UI.

### Patient-owned records & signing

The defining long-term vision ([explained in Core concepts](/docs/concepts#patient-owned-records-the-vision)):

- Records stored on the **patient's own device**, not only in the clinic database.
- Every clinician change **cryptographically signed**, blockchain-style.
- Changes stay **pending until the patient approves them** in a companion app.
- A **patient companion app** for reviewing and approving changes.

None of this is built yet. Today, accountability is provided by the
[activity log](/docs/guides/activity-analytics), which records every change with who
made it and when.

### Email verification enforcement

Verification emails are already sent (and the `/verify-email` page exists), but
sign-in currently does **not require** a verified address. Enforcement is wired and can
be switched on; it will be enabled by default once email delivery setup is documented
for production.

## Contributing

temetro is open source — if any of the above sounds like something you'd like to help
build, see the [contributing guide](/docs/developers/contributing). Wiring the real LLM
into the chat is a particularly good place to start.
