Temetro is being designed for hospitals that operate in multilingual environments. That means the documentation and product experience should be able to support more than six languages without becoming difficult to maintain.

## Why language support matters

Hospitals do not all operate in the same language, and some teams work across multiple languages in the same facility. Clear language support matters for:

- onboarding reception and records staff
- training administrators and IT teams
- reducing mistakes caused by unclear labels or instructions
- making self-hosting documentation easier to follow

## Supported languages roadmap

Temetro documentation is being prepared with support for more than six languages, including:

- English
- Arabic
- French
- Swahili
- Somali
- Amharic
- Hausa
- Yoruba
- Portuguese
- Afrikaans
- Oromo
- Tigrinya
- Zulu
- Xhosa
- Wolof
- Lingala

This list can expand over time as contributors help translate both the interface and the documentation.

## Documentation translation goals

The docs should eventually provide translated versions of the most important content first:

- installation steps
- environment variable explanations
- first-run instructions
- production deployment guidance
- authentication basics
- patient and appointment workflow overviews

This helps hospital IT teams and operational staff get value quickly, even before every page is fully translated.

## Product translation goals

For the main Temetro application, language support should focus on the areas staff see most often:

- navigation labels
- patient management actions
- appointment status language
- staff and role terminology
- settings and deployment labels
- authentication screens

This makes the core dashboard easier to adopt in real hospital environments.

## Suggested translation model

Temetro should keep translations organized in a way that is simple for contributors:

- one locale key per supported language
- shared terminology for clinical and administrative words
- review before publishing high-impact production text
- clear fallback behavior when a translation is incomplete

This keeps documentation and product text easier to grow over time.

## Recommended first rollout

A practical first rollout for multilingual docs would be:

1. English as the default source language
2. Arabic and French as the first translated docs
3. Swahili and Somali as the next operational priorities
4. Additional regional languages such as Oromo, Tigrinya, Zulu, Xhosa, Wolof, and Lingala added as translation capacity grows

This sequence gives Temetro a realistic path instead of trying to perfect every language at once.

## Contributing translations

If you want to help translate Temetro documentation, focus first on:

- accuracy of medical and administrative terminology
- clarity for non-developer hospital staff
- consistent wording across installation and deployment guides
- preserving step-by-step structure during translation

Translation contributions are especially valuable for hospitals where English-only documentation creates adoption barriers.
