ይህ መመሪያ ሶፍትዌር ልማት ባለሙያ ላልሆኑ የሆስፒታል IT ሰራተኞች የተጻፈ ነው። ዓላማውም Temetro ን ግምት ሳይጠቀሙ እንዲያሰማሩት ማገዝ ነው።

## Docker Compose ምን ያደርጋል?

Docker Compose ብዙ ተያያዥ አገልግሎቶችን በአንድ ጊዜ ያስጀምራል፦

- ለመተግበሪያ ውሂብ የ PostgreSQL የውሂብ ጎታ
- የ Temetro ዳሽቦርድ
- የዳሽቦርድ ውሂብ የሚያቀርብ የውስጥ Express API
- የ Temetro ሰነዶች ጣቢያ
- ለፕሮዳክሽን ትራፊክ አማራጭ NGINX reverse proxy

የ `api` አገልግሎቱ ብዙ ጊዜ በስታኩ ውስጥ ብቻ ይቆያል። የዳሽቦርድ frontend በ `INTERNAL_API_URL` ይደርስበታል።

እያንዳንዱን ክፍል በተናጠል ከመጫን ይልቅ አንድ ትእዛዝ ያሂዱ፣ Docker ደግሞ ስታኩን ያቀናጃል።

## መሠረታዊ የማስጀመሪያ ትእዛዝ

ከማሰማሪያ ፎልደሩ፦

```bash
docker compose up -d
```

ይህ የሚያደርገው፦

- `docker compose` Docker ን አሁን ባለው ፎልደር ውስጥ ያለውን Compose ፋይል እንዲጠቀም ያሳውቃል
- `up` አገልግሎቶቹን ያስጀምራል
- `-d` “detached mode” ማለት ነው፣ ስለዚህ ከበስተጀርባ እየሰራ ይቀጥላል

## ጠቃሚ ዕለታዊ ትእዛዞች

### እየሰሩ ያሉ አገልግሎቶችን አሳይ

```bash
docker compose ps
```

### የቀጥታ ሎጎችን ይመልከቱ

```bash
docker compose logs -f
```

### ስርዓቱን አቁም

```bash
docker compose down
```

### የተከማቸውን የውሂብ ጎታ ውሂብ አጥፋና አቁም

```bash
docker compose down -v
```

`-v` ን የውሂብ ጎታውን volume በእውነት ማጥፋት ሲፈልጉ ብቻ ይጠቀሙ፣ ለምሳሌ ፕሮዳክሽን ያልሆነ ዳግም ማስጀመር ሲያደርጉ።

## የፎልደር ተስፋዎች

የማሰማሪያ ፋይሎቹን የIT ቡድንዎ በቀላሉ ሊያገኛቸው በሚችል የተለየ ፎልደር ውስጥ ያስቀምጡ። የተለመደ አቀማመጥ ምናልባት እንዲህ ይሆናል፦

```text
/opt/temetro
```

ወይም በላፕቶፕ ላይ፦

```text
C:\Temetro
```

## ዝማኔዎች እንዴት ይሰራሉ?

አዲስ የማሰማሪያ ፋይሎች ስሪት ሲገኝ፦

1. የውሂብ ጎታዎን መጠባበቂያ ያድርጉ
2. የቅርብ የሪፖዚቶሪ ለውጦችን ያውርዱ
3. አገልግሎቶቹን እንደገና ይገንቡ እና ያስጀምሩ

ምሳሌ፦

```bash
git pull
docker compose up -d --build
```
