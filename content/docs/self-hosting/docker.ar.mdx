كُتب هذا الدليل لموظفي تقنية المعلومات في المستشفيات الذين قد لا يكونون مطورين برمجيين. الهدف هو مساعدتك على نشر Temetro دون تخمين.

## ماذا يفعل Docker Compose

يقوم Docker Compose بتشغيل عدة خدمات مترابطة معًا:

- قاعدة بيانات PostgreSQL لبيانات التطبيق
- لوحة Temetro
- واجهة Express داخلية لبيانات لوحة التحكم
- موقع توثيق Temetro
- وكيل NGINX عكسي اختياري لحركة الإنتاج

تبقى خدمة `api` عادة داخلية ضمن الحزمة، وتتصل بها واجهة لوحة التحكم عبر `INTERNAL_API_URL`.

بدلًا من تثبيت كل جزء يدويًا، تشغّل أمرًا واحدًا ويقوم Docker بتجميع الحزمة.

## أمر التشغيل الأساسي

من داخل مجلد النشر:

```bash
docker compose up -d
```

ما الذي يفعله ذلك:

- `docker compose` يطلب من Docker استخدام ملف Compose في المجلد الحالي
- `up` يشغّل الخدمات
- `-d` تعني “الوضع المنفصل” ليواصل العمل في الخلفية

## أوامر يومية مفيدة

### عرض الخدمات العاملة

```bash
docker compose ps
```

### عرض السجلات الحية

```bash
docker compose logs -f
```

### إيقاف النظام

```bash
docker compose down
```

### إيقاف النظام وحذف بيانات قاعدة البيانات المخزنة

```bash
docker compose down -v
```

استخدم `-v` فقط إذا كنت تنوي حذف وحدة تخزين قاعدة البيانات عمدًا، مثلًا أثناء إعادة ضبط غير إنتاجية.

## توقعات المجلدات

احتفظ بملفات النشر في مجلد مخصص يسهل على فريق تقنية المعلومات العثور عليه. قد يبدو الإعداد المعتاد مثل:

```text
/opt/temetro
```

أو على حاسوب محمول:

```text
C:\Temetro
```

## كيف تعمل التحديثات

عند توفر إصدار جديد من ملفات النشر:

1. انسخ قاعدة البيانات احتياطيًا
2. اسحب آخر التغييرات من المستودع
3. أعد بناء الخدمات وأعد تشغيلها

مثال:

```bash
git pull
docker compose up -d --build
```

## متى تنتقل إلى إعدادات الإنتاج

استخدم ملف Compose الخاص بالإنتاج وNGINX عندما:

- يستخدم الموظفون Temetro بانتظام من عدة أجهزة
- تحتاج إلى HTTPS
- تريد توجيهًا نظيفًا مثل `/docs`
- يجب أن يبقى الخادم يعمل طوال اليوم مع إعادة تشغيل تلقائية

للاطلاع على قائمة فحص الإنتاج الكاملة، تابع إلى [الإنتاج](./production).
