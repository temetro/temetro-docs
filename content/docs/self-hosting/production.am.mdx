የፕሮዳክሽን ማሰማሪያ ማለት ስርዓቱ ለመደበኛ አሰራር የሚታመንበት ደረጃ መድረሱ ማለት ነው። በዚያ ደረጃ ላይ እምነትና ደህንነት እንደመተግበሪያው ራሱ እኩል ጠቃሚ ናቸው።

## የፕሮዳክሽን ዝርዝር

ወደ ስራ ከማስገባትዎ በፊት የሚከተሉትን ሁሉ ያረጋግጡ፦

- ሰርቨሩ የተረጋጋ ኃይል እና በቂ ማቀዝቀዣ አለው
- ዕለታዊ ወይም በመርሐ ግብር የሚደረጉ የውሂብ ጎታ መጠባበቂያዎች ተዘጋጅተዋል
- `APP_URL` ሰራተኞች የሚከፍቱትን እውነተኛ hostname ይጠቀማል
- `BETTER_AUTH_SECRET` ጠንካራ የተፈጠረ እሴት ነው
- reverse proxy እና HTTPS ተዘጋጅተዋል
- የሰርቨሩ መዳረሻ ለተፈቀዱ አስተዳዳሪዎች ብቻ ተገድቧል

## የፕሮዳክሽን Compose override ይጠቀሙ

ስታኩን በዚህ ያስጀምሩ፦

```bash
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d
```

ይህ NGINX አገልግሎትን ይጨምራል እና ለፕሮዳክሽን የሚመች ባህሪን ይተገብራል።

## የ NGINX ኃላፊነቶች

በ Temetro የማሰማሪያ ስታክ ውስጥ NGINX የሚከተሉትን ይሰራል፦

- በ `80` እና `443` ፖርቶች መስማት
- ዳሽቦርዱን ወደ web አገልግሎት መስመር ማስተላለፍ
- `/docs` ን ወደ docs አገልግሎት መስመር ማስተላለፍ
- compression እና security headers አስተዳደር
- ቀላል `/health` endpoint ማቅረብ

## HTTPS

የታካሚ ወይም የሰራተኛ መረጃ በኔትወርክ ላይ ለሚተላለፍበት ማንኛውም አካባቢ HTTPS እንደ አስፈላጊ መቆጠር አለበት።

ሰርቲፍኬቶችን በ NGINX ቅንብር የሚጠቀምበት `ssl` ፎልደር ውስጥ ማስቀመጥ ይችላሉ። አንዳንድ ድርጅቶች የሚጠቀሙት፦

- በሆስፒታል IT የተሰጡ ውስጣዊ ሰርቲፍኬቶች
- በሰርቲፍኬት ባለስልጣን የተሰጡ የሕዝብ ሰርቲፍኬቶች
- ከ Temetro ስታክ ውጭ የሚተዳደሩ ራስ-ሰር የሰርቲፍኬት መሳሪያዎች

## የመጠባበቂያ ስትራቴጂ

ጥሩ የአነስተኛ ደረጃ የመጠባበቂያ እቅድ እነዚህን ያካትታል፦

- የሌሊት PostgreSQL dumps
- ሳምንታዊ ሙሉ የሰርቨር snapshot ወይም VM backup
- የቅርብ መጠባበቂያ የተደህነ ከመሣሪያ ውጭ ቅጂ
- በIT ሰራተኞች የተፈተነ የመመለሻ ሂደት

## የማሻሻያ ስትራቴጂ

ውስን የጥገና መስኮቶች ላላቸው ሆስፒታሎች፦

1. የውሂብ ጎታውን መጠባበቂያ ያድርጉ
2. ከስራ ሰዓት ውጭ የለውጥ ጊዜ ያቅዱ
3. የቅርብ ኮድ ያውርዱ
4. አገልግሎቶቹን እንደገና ይገንቡ
5. መግቢያን እና የታካሚ ዳሽቦርዱን ያረጋግጡ
