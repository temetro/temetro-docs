يساعد Temetro المستشفيات على إدارة المرضى والمواعيد والسجلات الطبية والموظفين من لوحة تحكم واحدة مستضافة ذاتيًا.

## لماذا Temetro؟

لا يزال كثير من برمجيات الرعاية الصحية في العديد من البيئات الإفريقية مرتفع الكلفة أو معتمدًا أكثر من اللازم على السحابة أو غير مناسب لواقع البنية التحتية المحلية. تم تصميم Temetro بطريقة مختلفة:

- **مجاني ومفتوح المصدر** حتى تتمكن المستشفيات من فحص البرنامج الذي تعتمد عليه وتعديله وامتلاكه.
- **مستضاف ذاتيًا** حتى تبقى بيانات المرضى داخل البنية التحتية التي تتحكم فيها المستشفى.
- **سهل النشر** باستخدام Docker Compose لفرق تقنية المعلومات الصغيرة.
- **عملي في البيئات محدودة الموارد** بواجهة نظيفة وبصمة تشغيلية صغيرة.

## بداية سريعة

```bash
git clone https://github.com/temetro/temetro-deploy
cd temetro-deploy
cp .env.example .env
docker compose up -d
```

افتح `http://localhost:3000` للوصول إلى لوحة التحكم و`http://localhost:3001/docs` لتصفح موقع التوثيق.

## لمن صُمم Temetro

يعد Temetro مناسبًا لـ:

- المستشفيات المحلية والإقليمية التي تريد لوحة داخلية خاصة
- المستشفيات التبشيرية والمنشآت غير الربحية ذات الميزانيات البرمجية المحدودة
- فرق تقنية المعلومات التي تفضل بنية بسيطة مستضافة ذاتيًا بدلًا من برمجيات الاشتراك
- المطورين والمنفذين الذين يحتاجون إلى أساس قابل للتوسعة

## ماذا تحصل عليه

يتضمن الإصدار الأولي:

- لوحة مستشفى متجاوبة مبنية بـ Next.js
- مصادقة بالبريد الإلكتروني وكلمة المرور مع Better Auth
- نماذج بيانات PostgreSQL وPrisma للمرضى والمواعيد وتدفقات عمل الموظفين
- صفحة هبوط عامة
- موقع توثيق كامل
- ملفات نشر Docker Compose مع توجيه NGINX
