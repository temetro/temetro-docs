# Security Policy

## Supported versions

Only the latest release on `main` receives security fixes.

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Email **khalidstudentxv@gmail.com** with:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Any suggested mitigations (optional)

You will receive an acknowledgement within 48 hours and a status update within 7 days.

## Scope

This repository is the temetro documentation site. Security reports are most relevant for:

- XSS or content injection
- Exposed credentials or secrets in the codebase
- Third-party script supply-chain issues

For vulnerabilities in the temetro product itself (auth, patient data), report to the [main repo](https://github.com/temetro/temetro).
